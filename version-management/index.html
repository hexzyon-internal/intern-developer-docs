Protocol Standard

The VERSION File Format

The VERSION file must contain only the semantic version number in a strictly standardized format. This file acts as the single source of truth for the entire build pipeline.

1.0.0

Critical Formatting Rules

  • No v prefix in the file (e.g., use 1.0.0, not v1.0.0)
  • No extra whitespace, newlines, or hidden characters
  • Must follow strict Semantic Versioning: MAJOR.MINOR.PATCH

Version Bump Decision Tree

Use this logic flow to determine the correct component to increment.

Did you make breaking changes?
 ├─ YES → Bump MAJOR (1.0.0 → 2.0.0)
 └─ NO
    ├─ Did you add new features?
    │  └─ YES → Bump MINOR (1.0.0 → 1.1.0)
    └─ NO
       └─ Did you fix bugs?
          └─ YES → Bump PATCH (1.0.0 → 1.0.1)

Update Scenarios

Scenario 1: Bug Fix

1.2.3 → 1.2.4 

echo "1.2.4" > VERSION
git add VERSION
git commit -m "fix: resolve authentication issue"
git push origin sync

Result: Image tagged as ghcr.io/username/project:v1.2.4

Scenario 2: New Feature

1.2.4 → 1.3.0 

echo "1.3.0" > VERSION
git add VERSION
git commit -m "feat: add user dashboard"
git push origin sync

Result: Image tagged as ghcr.io/username/project:v1.3.0

Real-World Examples

Mobile App Project

v1.0.0 → Initial release v1.0.1 → Fix login crash v1.0.2 → Fix payment processing v1.1.0 → Add dark mode v1.2.0 → Add social sharing v2.0.0 → Complete UI redesign (breaking)

REST API Service

v1.0.0 → Initial API release v1.0.1 → Fix response format v1.1.0 → Add /users endpoint v2.0.0 → Change auth from JWT to OAuth (breaking)

Docker Image Tag Examples

Single Repository

VERSION file: 1.2.3

ghcr.io/username/my-app:v1.2.3
ghcr.io/username/my-app:latest

Docker Compose Usage

services:
  app:
    image: ghcr.io/username/my-app:v1.2.3

Troubleshooting

Error: Workflow doesn't read VERSION

If the workflow fails with "VERSION file not found", verify file presence in root.

# Check file
ls -la VERSION

# Create if missing
echo "1.0.0" > VERSION

Error: Incorrect Tagging (v.1.0.0)

If tags appear malformed, strip invisible characters.

cat VERSION | tr -d '[:space:]' > VERSION.tmp && mv VERSION.tmp VERSION

Component Format Trigger Example
